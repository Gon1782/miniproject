팀소개

팀의 특징

  • 의사소통이 원활하고 분위기가 좋습니다.

    • - Git pull을 안하고 Git push를 해서
    이전에 수정했던 코드가 사라진 사건을
    대화를 통해 30분만에 해결한적이 있습니다.

    - 디자인 코드 등등을 서로서로 피드백을 확실히 해줘서
    더 나은 결과물을 도출해내며, 누군가의 어려움을
    다 같이 고민하여 극복해나갔었습니다.

  • 합리적으로 역할을 정합니다.

    • - 다들 중요한 역할을 맡고싶어했으나, 배려심이 넘쳐
    서로 양보하다가 오래전부터 내려오던 확실한 방법인
    사다리로 결정됐습니다.(팀장,발표,백엔드)

  • 항상 배가 고픈 팀

    • - 실제로 배가고픈 사람들이 많습니다.
    - 프로젝트를 다 완성해놓고 만족하지못해,
    더 많은 기능들을 추구합니다.

팀이 추구하는 목표

  1. 끊임없이 공부하고 다른팀원들과 공유하며
    팀원의 전체적인 실력향상을 추구하는 팀
  2. 원할한 커뮤니케이션에서 오는 피드백을통해
    서로서로 성장할수있도록 도와주는 팀
  3. 어떤 문제에 직면했을때 함께 고민해보고,
    집단지성을 이용해 해결할수 있는 팀
  4. 엄청나게 어려운문제에 가로막혔을때에도
    Never Give up 을 외치는 팀

우리 팀의 약속

  • 시간약속 잘지키기
  • 자기가 맡은일은 끝까지 완수하기
  • 서로 배려하기
  • 먼저 구글링하고 질문하기
  • 서로 피드백 해주기
  • 부정적인 태도 취하지 않기
  • 오류나도 짜증내지 않기
